# Explicit Wiring

Forge uses explicit dependency wiring instead of a service registry. All construction and injection happens in `pkg/app/wire.go`, which is generated by `forge generate`. There are no `init()` functions, no global registries, and no import side effects.

## Overview

The wiring system provides:
- **Explicit construction** — all dependencies are visible in one file
- **Constructor injection** — `Deps` structs with typed fields
- **Config-driven** — the config proto drives what gets built
- **Test harness** — generated test helpers with mock wiring
- **No magic** — no reflection, no `init()`, no hidden registration

## How It Works

### The Wiring File

`pkg/app/wire.go` is generated by `forge generate` and contains the full dependency graph:

```go
// pkg/app/wire.go — GENERATED by forge generate
package app

import (
    "github.com/myorg/myproject/internal/email"
    usersservice "github.com/myorg/myproject/services/users"
    configv1 "github.com/myorg/myproject/gen/config/v1"
)

type Deps struct {
    DB     *pgxpool.Pool
    Logger *slog.Logger
    Email  email.Contract
}

func BuildDeps(cfg *configv1.Config) (*Deps, error) {
    pool, err := pgxpool.New(ctx, cfg.DatabaseUrl)
    if err != nil {
        return nil, err
    }

    logger := slog.New(slog.NewJSONHandler(os.Stdout, nil))

    emailSvc := email.New(email.Deps{
        Logger: logger,
    })

    return &Deps{
        DB:     pool,
        Logger: logger,
        Email:  emailSvc,
    }, nil
}

func BuildServices(deps *Deps) []Service {
    return []Service{
        usersservice.New(usersservice.Deps{
            DB:     deps.DB,
            Logger: deps.Logger,
            Email:  deps.Email,
        }),
    }
}
```

### Constructor Injection Pattern

Every service and internal package follows the same pattern:

```go
// services/users/service.go
package usersservice

type Deps struct {
    DB     *pgxpool.Pool
    Logger *slog.Logger
    Email  email.Contract
}

type Service struct {
    deps Deps
}

func New(deps Deps) *Service {
    return &Service{deps: deps}
}

func (s *Service) Name() string {
    return "UsersService"
}

func (s *Service) Register(mux *http.ServeMux, opts ...connect.HandlerOption) {
    path, handler := usersv1connect.NewUsersServiceHandler(s, opts...)
    mux.Handle(path, handler)
}
```

### Config Proto

The config proto is the executable's instantiation contract:

```protobuf
// proto/config/v1/config.proto
message Config {
    string database_url = 1;
    string log_level = 2;
    int32 port = 3;
    EmailConfig email = 4;
}
```

This drives what gets constructed. Environment variables and flags map to config fields.

## Test Harness

The generated `pkg/app/testing.go` provides test helpers that wire up services with mock dependencies:

```go
// pkg/app/testing.go — GENERATED
package app

type TestMocks struct {
    Email *email.MockContract
    DB    *pgxpool.Pool  // test database or mock
}

func NewTestUsersService(t *testing.T) (*usersservice.Service, *TestMocks) {
    mocks := &TestMocks{
        Email: email.NewMockContract(t),
    }

    svc := usersservice.New(usersservice.Deps{
        Logger: slog.New(slog.NewTextHandler(io.Discard, nil)),
        Email:  mocks.Email,
    })

    return svc, mocks
}

func NewTestUsersServiceServer(t *testing.T) (usersv1connect.UsersServiceClient, func()) {
    svc, _ := NewTestUsersService(t)
    mux := http.NewServeMux()
    svc.Register(mux)
    server := httptest.NewServer(mux)
    client := usersv1connect.NewUsersServiceClient(http.DefaultClient, server.URL)
    return client, server.Close
}
```

### Usage in Tests

```go
func TestCreateUser(t *testing.T) {
    svc, mocks := app.NewTestUsersService(t)

    mocks.Email.On("Send", mock.Anything, "user@test.com", mock.Anything, mock.Anything).Return(nil)

    resp, err := svc.CreateUser(ctx, connect.NewRequest(&usersv1.CreateUserRequest{
        Email: "user@test.com",
    }))
    require.NoError(t, err)
    assert.NotEmpty(t, resp.Msg.Id)
}
```

## How Services Start

The generated `cmd/server.go` reads the config, builds dependencies, and starts the server:

```go
// cmd/server.go — GENERATED
var serverCmd = &cobra.Command{
    Use:   "server [services...]",
    Short: "Start the HTTP server",
    RunE: func(cmd *cobra.Command, args []string) error {
        cfg := loadConfig()
        deps, err := app.BuildDeps(cfg)
        if err != nil {
            return err
        }

        services := app.BuildServices(deps)
        mux := http.NewServeMux()

        // Register requested services (or all if none specified)
        for _, svc := range services {
            if len(args) == 0 || contains(args, svc.Name()) {
                svc.Register(mux, interceptors...)
            }
        }

        return http.ListenAndServe(fmt.Sprintf(":%d", cfg.Port), mux)
    },
}
```

## Adding New Dependencies

When you add a service (`forge add service`) or package (`forge package new`), `forge generate` updates `pkg/app/wire.go` to include the new dependency in the construction graph.
