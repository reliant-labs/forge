# Core Concepts

Forge is built on a set of core principles that guide its design and usage. Understanding these concepts is essential to getting the most out of the framework.

## Two Contract Systems

### Proto Contracts for External Boundaries

In Forge, **proto files are the source of truth for external service contracts**. This means:

1. **Define services in proto first**
   ```protobuf
   service UserService {
     rpc GetUser(GetUserRequest) returns (User);
   }
   ```

2. **Generate code from proto**
   ```bash
   forge generate
   ```

3. **Implement the generated interface**
   ```go
   func (s *Service) GetUser(ctx context.Context, req *connect.Request[...]) (*connect.Response[...], error) {
       // Your business logic
   }
   ```

### Go Interface Contracts for Internal Boundaries

Internal packages use Go interfaces defined in `contract.go`:

1. **Define the interface**
   ```go
   // internal/email/contract.go
   type Contract interface {
       Send(ctx context.Context, to, subject, body string) error
   }
   ```

2. **Implement the interface**
   ```go
   // internal/email/service.go
   type Service struct { ... }
   func (s *Service) Send(ctx context.Context, to, subject, body string) error { ... }
   ```

3. **Generate mocks and middleware**
   ```bash
   forge generate
   ```

### Why Two Systems?

| Need | Proto | Go Interface |
|------|-------|-------------|
| Cross-language support | Yes | No |
| Wire format | Yes | N/A |
| Channels | No | Yes |
| Complex Go types | No | Yes |
| Factories, closures | No | Yes |
| Variadic arguments | No | Yes |

### Proto Directory Structure

```
proto/
├── config/v1/     # Config — instantiation contract
├── services/      # Service RPCs (external boundary)
│   └── user/
│       └── v1/
│           └── service.proto
└── db/            # Database entities (deprecated, ORM-owned)
    └── models.proto
```

## Single Binary Design

Every Forge project produces a **single Go binary** with a Cobra CLI:

```bash
myproject server                    # Start all services
myproject server api users          # Start specific services
myproject version                   # Build info
```

All services share one process, one HTTP mux, one middleware stack. No per-service binaries.

## Explicit Wiring

All dependencies are constructed in `pkg/app/wire.go`, generated by `forge generate`:

```go
// pkg/app/wire.go — GENERATED
func BuildDeps(cfg *configv1.Config) (*Deps, error) {
    pool, err := pgxpool.New(ctx, cfg.DatabaseUrl)
    emailSvc := email.New(email.Deps{Logger: logger})

    return &Deps{
        DB:    pool,
        Email: emailSvc,
    }, nil
}
```

No `init()` functions, no registries, no hidden side effects.

## Constructor Injection

Services declare dependencies explicitly in `Deps` structs:

```go
type Deps struct {
    DB    *pgxpool.Pool
    Email email.Contract  // Go interface from internal/email/contract.go
}

func New(deps Deps) *Service {
    return &Service{deps: deps}
}
```

### Benefits

- **Explicit dependencies**: No hidden dependencies or global state
- **Testability**: Easy to inject mocks in tests
- **Type safety**: Compile-time checking
- **Visibility**: All wiring in one generated file

## Contract Enforcement

The `forge lint --contract` command enforces that all exported methods on service types match their contract:

- **Proto API services**: Methods must correspond to proto RPCs
- **Internal packages**: Methods must be in the `contract.go` interface
- **Exported types, functions, constants**: Unrestricted (not checked)

## Test Harness

Generated test helpers in `pkg/app/testing.go`:

```go
func TestCreateUser(t *testing.T) {
    svc, mocks := app.NewTestUsersService(t)
    mocks.Email.On("Send", mock.Anything, mock.Anything, mock.Anything, mock.Anything).Return(nil)

    resp, err := svc.CreateUser(ctx, connect.NewRequest(&usersv1.CreateUserRequest{
        Email: "user@test.com",
    }))
    require.NoError(t, err)
}
```

## Error Handling

Forge uses Connect's error model:

```go
import "connectrpc.com/connect"

func (s *Service) GetUser(ctx context.Context, req *connect.Request[usersv1.GetUserRequest]) (*connect.Response[usersv1.User], error) {
    if req.Msg.Id == "" {
        return nil, connect.NewError(connect.CodeInvalidArgument, errors.New("user ID is required"))
    }

    user, err := s.deps.DB.GetUser(ctx, req.Msg.Id)
    if errors.Is(err, database.ErrNotFound) {
        return nil, connect.NewError(connect.CodeNotFound, fmt.Errorf("user %s not found", req.Msg.Id))
    }
    if err != nil {
        return nil, connect.NewError(connect.CodeInternal, fmt.Errorf("database error: %w", err))
    }

    return connect.NewResponse(user), nil
}
```

### Error Codes

| Code | When to Use |
|------|-------------|
| `CodeInvalidArgument` | Bad request parameters |
| `CodeNotFound` | Resource doesn't exist |
| `CodeAlreadyExists` | Resource already exists |
| `CodePermissionDenied` | Not authorized |
| `CodeUnauthenticated` | Not authenticated |
| `CodeResourceExhausted` | Rate limited or quota exceeded |
| `CodeInternal` | Internal server error |
