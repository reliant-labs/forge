# Service Layer

Every domain in a forge project has the same shape:

```
internal/things/
  contract.go    # Service interface — the public surface
  service.go     # implementation behind the interface
  errors.go      # domain-typed error sentinels and types
  mock_gen.go    # generated by `forge generate`
```

Per-package observability wrappers (`middleware_gen.go`, `tracing_gen.go`, `metrics_gen.go`) were removed in 1.7. Logging, tracing, metrics, recovery, and request-id are applied uniformly at the Connect handler boundary via `forge/pkg/observe` interceptors (`observe.DefaultMiddlewares(deps)` in `cmd/server.go`). Per-method opt-in helpers — `observe.LogCall` / `observe.TraceCall` / `observe.NewCallMetrics` — are available for the rare case of an explicit child span at a service-to-service call site.

The handler layer (`handlers/<svc>/`) is wire glue; this layer is the actual app. See `api/handlers` for the other half.

## The Service interface

`contract.go` declares what the package does, in the language of the domain — not in proto types, not in DB row types.

```go
// internal/things/contract.go
package things

import (
    "context"
    "time"
)

type Service interface {
    DoThing(ctx context.Context, in DoThingInput) (DoThingResult, error)
    GetThing(ctx context.Context, id string) (Thing, error)
    ListThings(ctx context.Context, in ListThingsInput) (ListThingsResult, error)
}

type Thing struct {
    ID        string
    Name      string
    OwnerID   string
    CreatedAt time.Time
    UpdatedAt time.Time
}

type DoThingInput struct {
    UserID string
    Name   string
}

type DoThingResult struct {
    ID        string
    CreatedAt time.Time
}

type ListThingsInput struct {
    OwnerID  string
    PageSize int
    Cursor   string
}

type ListThingsResult struct {
    Items      []Thing
    NextCursor string
}
```

A few rules pay off immediately:

- **One input struct per method**, even when it has one field today. Adding a second field tomorrow doesn't break call sites.
- **Domain types, not proto types.** No `*apiv1.Thing` here, no `*db.Thing` either. The service is the seam between wire and storage.
- **Plain `time.Time`, plain `string` IDs.** Wrap in `*timestamppb.Timestamp` at the handler boundary; convert to/from `pgtype.UUID` at the storage boundary.
- **`Service` is the canonical name** for single-impl. Use `<Name>er` (`Storer`, `Authenticator`) only for multi-impl strategies. `forge generate` assumes `Service`.

`forge generate` reads `contract.go` and emits `mock_gen.go` — your test seam, free. Observability is applied at the Connect handler boundary by `forge/pkg/observe` interceptors, not as per-package codegen wrappers.

## The implementation

```go
// internal/things/service.go
package things

import (
    "context"
    "errors"
    "fmt"
    "time"

    "github.com/example/proj/internal/db"
)

type Deps struct {
    DB     *db.Queries
    Now    func() time.Time
    NewID  func() string
}

type svc struct {
    deps Deps
}

func New(deps Deps) Service {
    if deps.Now == nil {
        deps.Now = time.Now
    }
    if deps.NewID == nil {
        deps.NewID = newULID
    }
    return &svc{deps: deps}
}

func (s *svc) DoThing(ctx context.Context, in DoThingInput) (DoThingResult, error) {
    if in.Name == "" {
        return DoThingResult{}, ValidationError{Field: "name", Reason: "required"}
    }

    row, err := s.deps.DB.InsertThing(ctx, &db.Thing{
        ID:        s.deps.NewID(),
        Name:      in.Name,
        OwnerID:   in.UserID,
        CreatedAt: s.deps.Now(),
    })
    if err != nil {
        if db.IsUniqueViolation(err) {
            return DoThingResult{}, ErrAlreadyExists
        }
        return DoThingResult{}, fmt.Errorf("insert thing: %w", err)
    }

    return DoThingResult{ID: row.ID, CreatedAt: row.CreatedAt}, nil
}

func (s *svc) GetThing(ctx context.Context, id string) (Thing, error) {
    row, err := s.deps.DB.GetThing(ctx, id)
    if err != nil {
        if errors.Is(err, db.ErrNoRows) {
            return Thing{}, ErrNotFound
        }
        return Thing{}, fmt.Errorf("get thing: %w", err)
    }
    return toDomain(row), nil
}

// ... ListThings, etc.
```

Notes on the shape:

- **`Deps` struct, not positional args.** Adding a dep doesn't churn every call site, and `Deps{DB: ..., Now: time.Now}` is self-documenting.
- **Constructor returns the interface.** `New(Deps) Service`. Consumers depend on `things.Service`, never on `*things.svc`.
- **Inject `time.Now` and ID generators.** Tests get deterministic timestamps and IDs without monkey-patching.
- **Domain errors flow up.** Convert DB-driver errors (`db.ErrNoRows`, unique-violation codes) to your sentinel set at the storage boundary.

## Error domain types

Use `forge/pkg/svcerr` sentinels and constructors directly — do not redeclare `ErrNotFound` / `ErrAlreadyExists` / etc. per package. The shared sentinel set is what makes `svcerr.Wrap(err)` work uniformly in every handler. Reach for a custom error type only when the failure carries structured detail beyond what the sentinel categories express.

```go
// internal/things/contract.go (or service.go)
package things

import (
    "context"
    "errors"
    "fmt"

    "github.com/reliant-labs/forge/pkg/svcerr"
)

func (s *svc) GetThing(ctx context.Context, id string) (Thing, error) {
    row, err := s.deps.DB.GetThing(ctx, id)
    if err != nil {
        if errors.Is(err, db.ErrNoRows) {
            return Thing{}, svcerr.NotFound("thing")
        }
        return Thing{}, fmt.Errorf("get thing: %w", err)
    }
    return toDomain(row), nil
}
```

For domain-specific structured detail (e.g., a validation failure naming the field), keep the typed-error pattern AND wrap a sentinel so `svcerr.Wrap` still maps to the right Connect code:

```go
// internal/things/errors.go
package things

import (
    "fmt"

    "github.com/reliant-labs/forge/pkg/svcerr"
)

// ValidationError carries field-level detail for client display.
// Wraps svcerr.ErrInvalidArgument so `svcerr.Wrap` maps it to
// connect.CodeInvalidArgument automatically.
type ValidationError struct {
    Field  string
    Reason string
}

func (e ValidationError) Error() string {
    return fmt.Sprintf("validation: %s: %s", e.Field, e.Reason)
}

func (e ValidationError) Unwrap() error { return svcerr.ErrInvalidArgument }
```

Notes:

- **Sentinels** are the `svcerr.Err*` set; service authors write `return nil, svcerr.NotFound("user")` rather than declaring their own `ErrNotFound`.
- **Typed errors** (`errors.As`-checkable) are for cases that need extra detail — implement `Unwrap() error` returning the matching `svcerr.Err*` so the handler-side `svcerr.Wrap` maps it correctly.
- **Wrap upstream errors** with `fmt.Errorf("...: %w", err)` to add context. Never replace.
- **No per-service `errors.go` boilerplate.** Drop the file unless you have typed errors with structured detail.

## Wiring in `pkg/app/setup.go`

```go
// pkg/app/setup.go (your code — never edit bootstrap.go)
func Setup(app *App) error {
    app.Things = things.New(things.Deps{
        DB:    app.DB,
        Now:   time.Now,
        NewID: ulid.Make,
    })
    return nil
}
```

Observability — logging, tracing, metrics, recovery, request-id — is applied at the handler boundary by the `observe.DefaultMiddlewares(deps)` interceptor chain wired in `cmd/server.go`. The service body and `setup.go` carry no observability code.

## Testing

The service is the natural unit-test boundary. Two patterns matter:

- **Pure logic (validation, transforms):** table-driven tests on the package directly. No mocks needed.
- **DB-touching paths:** integration tests against a real database — see `testing/integration`. The mock generated for `Service` is for *consumers* to use (handlers, other services), not for testing the service against itself.

```go
func TestThingsService_DoThing(t *testing.T) {
    t.Parallel()

    fixedTime := time.Date(2025, 1, 1, 0, 0, 0, 0, time.UTC)
    s := things.New(things.Deps{
        DB:    fakedb.New(t),
        Now:   func() time.Time { return fixedTime },
        NewID: func() string { return "thing_01" },
    })

    res, err := s.DoThing(ctx, things.DoThingInput{UserID: "u1", Name: "n"})
    if err != nil {
        t.Fatalf("unexpected: %v", err)
    }
    if res.ID != "thing_01" || !res.CreatedAt.Equal(fixedTime) {
        t.Fatalf("got %+v", res)
    }
}
```

When a handler test wants to drive the handler's own logic without exercising the service, use `things.MockService` (from `mock_gen.go`). See `testing/patterns` Pattern 1 and the `contracts` skill.

## Multi-method services

For a domain with many methods, keep them in `contract.go` as long as the methods all belong to the same domain. Splitting `Service` into `Reader` + `Writer` interfaces is a sometimes-useful refinement (callers that only read can depend on `Reader`), but start with one `Service` and split when a real call site needs the narrower interface.

## Rules

- `internal/<svc>/contract.go` declares the `Service` interface in domain terms, not proto or DB terms.
- `internal/<svc>/service.go` implements it; constructor returns the interface.
- `Deps` struct for dependencies; inject `time.Now` and ID generators.
- Use `forge/pkg/svcerr` sentinels (`svcerr.NotFound("user")`, `svcerr.PermissionDenied("admin only")`, …) directly — do not redeclare `ErrNotFound` / `ErrAlreadyExists` per package. Custom typed errors should `Unwrap() error` to a `svcerr.Err*` sentinel.
- Wrap upstream errors with `fmt.Errorf("...: %w", err)`. Convert driver/storage errors to `svcerr` sentinels at the storage edge.
- Never import a proto package (`apiv1`) from a service. Translation is the handler's job.
- Never hand-edit `mock_gen.go`. Edit `contract.go` and re-run `forge generate`.
- Wire concrete dependencies in `pkg/app/setup.go`. Bootstrap.go is generated. Observability is applied at the handler boundary by `forge/pkg/observe` interceptors.

## Marking optional Deps fields

Required deps live in `validateDeps()` so they fail fast at construction. **Optional deps** — fields a service can run without (a NATS publisher used only on the rollback path, an audit fallback, an optional gateway feature) — are tagged with `// forge:optional-dep` on the line directly above the field:

```go
type Deps struct {
    Logger *slog.Logger
    Config *config.Config
    Repo   Repository

    // NATSPublisher publishes domain events; nil disables rollback.
    // forge:optional-dep
    NATSPublisher EventPublisher
}
```

The marker tells:

- **wire_gen** — emit the typed-zero silently. No `// TODO: wire X`. No `UNRESOLVED FIELDS` header entry.
- **validateDeps()** — do NOT add a check. The marker says "nil is OK".
- **the upgrade codemod** — leave per-RPC `if s.deps.X != nil { ... }` guards alone.

Misplaced markers (on the struct itself, on a method docstring, etc.) are caught by `forge lint --conventions` (`forgeconv-optional-dep-marker-position`). Unresolved non-optional deps are surfaced by `forge lint --wire-coverage`.

## When this skill is not enough

- **Choosing whether a package needs a contract** at all (pure utilities) — see `contracts`.
- **The handler half** (validation, `svcerr.Wrap`, proto↔internal conversion) — see `api/handlers`.
- **Multiple implementations / strategy pattern** — see `contracts`, "Multi-impl with strategy pattern".
- **Cross-service orchestration** — keep the orchestration in one service that depends on the others' `Service` interfaces; do not orchestrate in handlers or in `pkg/app/setup.go`.
- **DB schema and ORM details** — see `db`.
- **Naming conventions** for the `Service` / `Deps` / `New` triple, `*App` field casing, and snake_case directory paths — see `architecture` → **Naming conventions**.
